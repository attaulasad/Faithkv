## Seeds

Three global seeds, frozen: `13, 42, 2026` (`configs/lock.yaml`). Per-example
seeds are derived deterministically and order-independently via
`kvcot.utils.seeding.derive_seed(global_seed, dataset_name, problem_index)`
— SHA-256 of a canonical string, never Python's randomized `hash()`, never
dict/iteration order. FullKV and R-KV receive the identical derived seed for
the same example (the function takes no condition parameter — see
`tests/unit/test_seeding.py::test_full_kv_and_rkv_share_seed_derivation`).

Manifest sampling uses a separate, also-fixed seed: `13` (Python's
`random.Random(13).sample(...)`, `kvcot.data.freeze_gsm8k_manifest`).

Bootstrap resampling (`kvcot.analysis.stats.bootstrap_ci_mean`) uses its own
fixed seed, `20260715`, via `numpy.random.default_rng` — not the global
`numpy.random` state, so it cannot be perturbed by unrelated code running
earlier in the same process.

## Determinism on GPU

`torch.use_deterministic_algorithms(True)` and
`CUBLAS_WORKSPACE_CONFIG=:4096:8` are set in every GPU test
(`tests/integration/test_replay_gpu.py::_set_determinism`).
FlashAttention-2 forward determinism at fixed shapes is `[unverified]` —
tested against both `flash_attention_2` and `sdpa`, with only discrete
event/token identity treated as a hard gate (`docs/REPLAY_DESIGN.md` §5).

## Reproducing the frozen manifests

```bash
pip install -e ".[cpu-tools]"
kvcot freeze-manifests --config configs/lock.yaml
```

This is idempotent: record IDs are deterministic
(`manifest-{dataset}-{source_row_index}`), so re-running with the same seed
reproduces byte-identical rows.
